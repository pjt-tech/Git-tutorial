인덱스란 데이터에 색인을 표시하는것

인덱스 장.단점 
- 장점 : 검색 속도가 빨라지며 해당 쿼리에 부하가 줄어 시스템 성능이 향상
- 단점 : 인덱스 저장공간필요 (전체 시스템의 10%)
          데이터변경작업이 많으면 오히려 성능이 나빠짐, 인덱스생성에 많은시간필요

인덱스 종류와설명
- 클러스터형 인덱스 : 영어사전처럼 내용이 순서대로 정렬되어 있어 인덱스가 책의 내용과 같음,테이블당 하나만 생성가능
		   행 데이터를 인덱스로 지정한 열에 맞춰서 자동으로 정렬함. (primary key)
- 보조 인덱스 : 찾아보기에서 먼저 단어를 찾은 후 그옆에 표시된 페이지로 이동하여 원하는 내용을 찾는 것과 같은 개념
	       테이블당 여러개 생성가능. (unique)

인덱스 비트리 (클러스터형 인덱스) ,테이블 어떻게 만들수있는지 구성

클러스터형 인덱스를 만들려면 무엇을 적으면 클러스터형이 된다고함
- 테이블 생성시 Primary key 를 설정하면 해당 열에 클러스터형 인덱스가 생성됨
- alter table table명 add constraint 키의 별칭 primary key (컬럼명)
- unique not null

고유보조인덱스 만들기
-보조인덱스생성 : create index idx_테이블_컬럼 on 테이블명 (인덱스로 지정할 컬럼);
-고유 보조인덱스 생생 : create unique index idx_테이블_컬럼 on 테이블명 (인덱스로 지정할 컬럼);

인덱스로 두개의 열을 조합하여 인덱스 생성 후 삭제 
- 두개의 열을 조합한 인덱스 생성 : create index idx_테이블_컬럼_컬럼 on 테이블명(인덱스컬럼,인덱스컬럼);
- 생성한 인덱스 삭제 : Drop index 인덱스이름 on 테이블명 ;
