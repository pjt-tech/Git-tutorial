관계형 데이터베이스 관리시스템 서술형

- 가장 보편화된 데이터베이스관리시스템. 데이터를 저장하는 테이블의 일부를 다른 테이블과 상하 관계로 표시하며 상관관계를 정리함
  변화하는 업무나 데이터 구조에 대한 유연성이 좋아 유지관리가 용이함
  관리시스템으로는 oracle,sql server, mysql, mariadb 등이 있다.

DBMS의 특징

- 데이터 무결성 : 부적절한 자료가 입력되어 동일안 내용에 대하여 서로 다른 데이터가 저장되는 것을 허용하지않음
- 데이터 일관성 : 삽입,삭제,갱신,생성 후에도 저장된 데이터는 변함없이 일정
- 데이터 회복성 : 장애가 발생시 특정상태로 복구되어야하는 성질
- 데이터 보안성 : 불법적인것들로부터 보호되어야 하는 성질
- 데이터 효율성 : 응답시간 ,저장공간 활용 등이 최적화 되어 사용자, 소프트웨어 , 시스템 등의 요구 조건을 만족 시켜야 하는 성질

데이터 입력 , 수정 ,삭제 , 검색 쿼리  

osi 7계층 
-응용 계층 - 사용자 친화 환경 제공 호스트
-표현 계층 - 코드 문자 등을 번역하여 일관되게 전송하고 압축,해제,보안 기능도 담당 호스트
-세션 계층 - 송신,수신 간의 논리적 연결 호스트
-전송 계층 - 송신,수신 프로세스 간의 연결 tcp/ip udp
-네트워크 계층 - 다수의 중개 시스템 중 올바른 경로를 선택하도록 지원 라우터
-데이터 링크 계층 - 오류와 흐름을 제거하여 신뢰성 있는 데이터를 전송 브리지,스위치
-물리계층 - 실제 장비를 연결 허브 리피터
 
ARP-네트워크 계층 주소를 물리주소로 변환하기 위해 사용 ip주소를 mac주소로

TCP- 수신 측의 수신상태,수신여부를 단계별로 체크해가며 데이터를 전송 , 대표서비스 : ftp,http,smtp등

UDP- 망으로 데이터를 송신할 뿐 확인작업 없음 ,하나의 송신정보를 다수가 수신해야 할 경우 사용 대표서비스 : DNS,NFS 등

허브 장점,단점
하나의 네트워크 데이터 송신을 여러개의 포트로 나누어 뿌려준다 
단점은 아무리 빠른 속도를 쓴다 해도 어느 한쪽이 통신중이면 나머지는 대기해야함

리피터 : 디지털 신호를 증폭시켜줌

스위치와 라우팅

스위치는 허브는 나누지 못하는 콜리전 도메인을 나눌 수 있음 1.2번에서 통신이생기면 나머지는 기다려야하는 허브와는 달리 스위치는 나머지 통신도 가능하다 
브리지는 소프트웨어 기반, 속도가 느림 , 포트들이 같은속도를 지원

라우터 정의 - 자신이 가야 할 길을 자동으로 찾아서 갈수 있는 능력을 가진것
경로결정 - 데이터 패킷이 목적지까지 갈 수 있는 길을 검사하고 어떤 길로 가는 것이 가장 적절한지 결정
스위칭 - 길이 결정되면 그 쪽으로 데이터 패킷을 스위칭해줌
가장 좋은 경로를 결정하기위해 라우팅 알고리즘을 사용하고 라우팅 테이블을 만들어서 관리함
